## Part 1 — Weekly Reporting

### Format choice: Markdown (.md)

**Why markdown over PDF/Excel:**
- Auto-generated by GitHub Actions after each weekly scrape
- Renders beautifully in GitHub web UI — click and read
- Copy-paste friendly for Instagram captions, blog posts, anywhere
- Zero extra dependencies, no Python libraries needed
- Free, automatic, version-controlled in git
- Readable on phone

**Format ranking:**
| Format | Verdict |
|---|---|
| 🥇 Markdown | Best balance: easy to generate, easy to read, easy to share |
| 🥈 HTML | Good for visual reports with charts, slightly more complex |
| 🥉 Excel | Familiar but harder to share via link, needs library |
| ❌ PDF | Overkill — heavy, not editable, not viewable on GitHub |

### What the weekly report contains

After every Monday scrape, GitHub Actions generates `reports/YYYY-MM-DD.md` with:

```markdown
# Tbilisi Real Estate Report — May 4, 2026

## Summary
- Active listings tracked: 5,127 (+317 from last week)
- New listings this week: 412
- Delisted this week: 95 (avg time on market: 38 days)
- Price drops this week: 67 listings (avg drop: 6.2%)

## Market Movement
| District | Price/m² | Change vs last week | Active listings |
|---|---|---|---|
| Vake | $2,650 | +1.2% | 287 |
| Saburtalo | $1,820 | -0.5% | 412 |
| ...

## Days on Market
| District | Avg DoM | Median DoM | Sample |
|---|---|---|---|
| Vake | 47 days | 32 days | 287 |
| ...

## Renovation Premium
| District | Premium % | Sample |
|---|---|---|
| Saburtalo | +28% | 142 |
| ...

## Top Movers
- Biggest price drops this week
- Newly listed (premium tier)
- Notable delistings
```

### Workflow

1. Open report on phone (link to `https://github.com/davita555-beep/georgia-realestate/blob/main/reports/YYYY-MM-DD.md`)
2. Read in 30 seconds
3. Screenshot juicy stats for Instagram
4. Lives forever in git → can compare May vs. August in 6 months

**This is a Phase 6 task** — to be built after Phase 5 (rents pipeline). One Python script appended to the GitHub Actions workflow that reads `listings.json` + `prices.json` and writes the markdown report.

## Part 2 — Public vs. Private Data Strategy

### Mental Model: 3 Buckets

**🌐 Bucket 1 — Public on web (free, drives SEO + lead gen)**

Show things that:
- Are useful to a casual visitor researching their apartment
- Build SEO authority for tbilisiprice.ge
- Make people share/screenshot/link
- Don't reveal your data moat

**🔒 Bucket 2 — Database only (your moat)**

Keep things that:
- Took weeks/months to accumulate
- Would let competitors copy your edge if exposed
- Are too granular to be useful to retail visitors
- Could become a paid product later

**📊 Bucket 3 — Public but aggregated/teaser (builds trust)**

Show things that:
- Demonstrate you have the data (builds trust + authority)
- But don't give away the full granularity
- Drive people to subscribe / DM / contact

### Applied to your current data

| Data | Show publicly? | Why |
|---|---|---|
| District average price/m² | ✅ Public | Already there, classic SEO content |
| Price/m² for new build vs resale | ✅ Public | Already there, useful for buyers |
| District price chart | ✅ Public | Already there |
| Mortgage calculator | ✅ Public | Already there |
| **Renovation premium % per district** | ✅ Public (aggregated) | Unique content — show high-level only |
| **By condition breakdown** (renovated/black/white frame) | 🟡 Public, current snapshot only | Show now, hide historical |
| **By project type breakdown** | 🟡 Public, partial | Show top 3 projects per district, not all 13 |
| **Average DoM per district** | ✅ Public (after 30+ days of data) | Killer SEO content once it's real |
| **Individual listing DoM** | ❌ Private — moat | Per-apartment "X days on market" is negotiation gold — keep for paid/leads |
| **Price change history per listing** | ❌ Private — moat | "This apartment dropped price 3 times" — premium signal |
| **Seller name analytics** (agency vs private) | ❌ Private — moat | Useful for agents — potential paid product |
| **Final delisted prices** | 🟡 Aggregated only | Show district avg "selling price", hide individual listings |
| **Listing-level full data (listings.json)** | ❌ Private always | Never expose. Your raw asset. |

### The Strategic Principle

> **Public = the headline. Private = the receipts.**

Show enough public data that people trust you have the goods.
Keep enough private that they need to come to you for the deep insights.

### Concrete Roadmap for tbilisiprice.ge

**Add to public site (next 2–4 weeks):**
1. Renovation premium chart per district
2. "By condition" price breakdown (per district selector)
3. "By project type" price breakdown (top 3 projects per district)
4. Once DoM has 30+ days of data → district-level avg DoM chart

**Keep strictly private (database only):**
1. Individual listing memory
2. Per-listing price history
3. Per-listing DoM
4. Seller analytics
5. Any per-listing-level data

**Lead gen mechanism (Instagram + DM):**
- Post aggregated stats publicly (e.g., "Vake apartments avg 47 days on market")
- DM with specific apartment-level intel when people comment "მინდა"
- Private data becomes the value-add for leads

### Why this works

1. **Public stats build trust.** Visitors see you have unique data → they bookmark, share, link.
2. **SEO authority compounds.** "Tbilisi renovation premium" "Vake days on market" — these are uncontested keywords. You'll dominate them.
3. **Private data drives conversion.** "We have apartment-level data — comment to get matched" = clear ask, clear value.
4. **Future paid tier exists.** When you launch a $X/month subscription, you have ready-made premium content (DoM per listing, price history, agent intel).

### Watch out for

- **Don't expose listing-level data accidentally** via API responses. Anything that returns a single listing's full record is a leak.
- **Watermark Instagram screenshots** with tbilisiprice.ge logo — competitors will copy your stats.
- **Rate-limit any public API endpoint** if you build one. Bots will scrape your scrape.
